MongoDB란? 
MongoDB 의 경우 NoSQL(Not only SQL) 의 하나로 비관계형 데이터를 저장하고 처리하는 오픈소스 데이터베이스 시스템 입니다.

MongoDB의 중요한 내용의 경우 아래의 네가지로 볼 수 있습니다.

닫기
1. 문서(Documentation) 지향 데이터 모델

 - 데이터를 BSON(Binary Json) 형태의 문서(Documentation) 로 저장하며, 필드 - 값 으로 매칭되어 구성이 됩니다. 문서는 collection에 저장이 되며, collection 은 database 에 저장됩니다.

 

2. 스키마의 유연성

 - 스키마를 미리 정의하지 않고 데이터를 저장할 수 있습니다. 

 

3. 분산 및 확장성

 - 수평 확장이 가능한 분산 아키텍쳐를 가지고 있기 때문에 대량의 데이터를 처리 할 수 있으며 높은 가용성을 유지할 수 있습니다. 클러스터를 구성하고 데이터를 분산 저장함으로써 성능과 확장성을 향상시킬 수 있습니다.

 

4. 인덱싱

 - 다양한 유형의 인덱스를 지원합니다.

 

 

MongoDB 와 RDB(Relational Database)의 차이
닫기
1. 데이터 모델 

  - RDB

       1. 테이블을 사용하여 데이터를 저장

       2. 각 테이블은 이미 정의된 Column의 집합에 의해 정의

  - MongoDB 

       1. 문서 기반의 데이터 모델을 사용

       2. 각 문서는 유연한 구조를 가짐

 

2. 스키마 유연성

  - RDB 

       1. 엄격한 스키마 정의가 필요한 경우가 많음.

       2. 새로운 필드를 추가하거나 변경하는 경우에는 테이블 구조 변경 필요.

  - MongoDB

       1. 동적인 스키마를 가지므로 필드를 자유롭게 추가하거나 수정가능

 

3. 조인

   - RDB

       1. 여러 테이블 간에 조인을 사용해서 데이터를 연결 할 수 있음

   - MongoDB

       1. 조인을 지원하지 않으며 중첩된 문서 구조나 참조를 통해서 데이터를 연결함.

 

4. 트랜젝션

   - RDB

       1. ACID(원자성, 일관성, 고립성, 지속성)의 원칙을 엄격하게 따르며 하여 데이터 일관성과 안정성을 보장함.

   - MongoDB 

       1. 트랜잭션은 일부 지원하며 다중 문서 트랜잭션과 스냅샷 격리 수준을 제공함.

 

5. 스케일링

   - RDB

       1. 수직확장 (Vertical scaling) 을 통해 성능을 향상시킴.

   - MongoDB

       1. 수평확장 (Horizontal scaling) 을 강조하여 대용량 데이터 처리와 고가용성을 강조함.
